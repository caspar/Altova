Installing the development environment

This outlines how to manually install your developer environment for the GUI automation. For a quicker, automated installation, please refer to the Quick start document. If everything worked there, this page is redundant.

Sun Java

Please download the latest version of Java JDK from Sun, and to avoid problems install only the JDK and source code bits. That means, choose advanced installation and avoid installing the JRE part. For this example (and for your future peace of mind) install to
c:\java\jdk1.6.0
(or whatever the latest version is), and not "Program Files".

Apache Ant

Download the latest version of Apache Ant as a ZIP file, and unpack it to
C:\java\apache-ant-1.7.0
(or whatever the latest version is).

JUnit

Download version 4.3 of JUnit as a ZIP file, and unpack it to
C:\java\junit4.3
.
Note: version 4.5 is not compatible

CVS

Get WinCVS from http://www.wincvs.org/ (or from the installation location of you colleague at
C:\Program Files\GNU\WinCvs 1.2
), and copy to
C:\java\WinCVS 1.2
.

Setting up your PATH

Now that you've installed all the basic software, you can add these to your system environment PATH variable, and also add the CVSROOT variable. Either press the Windows Key and Pause/Break, or Go Start -> Settings -> Control Panel -> System. Then in the Advanced tab, click the Environment Variables button at the bottom. Based on the paths and versions used in this example, you should add PATH = C:\java\apache-ant-1.7.0\bin;C:\java\jdk1.6.0\bin;C:\java\WinCvs 1.2; and CVSROOT = :pserver:your_windows_username@viepcvs01.vie.altova.com:/cvs/qa

Since you will need the junit-4.x.jar to be able to run or compile your project,you should also add/set in your system environment the following variable:
  • CLASSPATH= your Junit_Root_Path\junit-4.x.jar ( if case you used the paths suggested in this tutorial, CLASSPATH=C:\java\junit4.3\junit-4.3.jar).
Also note that this step includes some variables for the robot test cases. Please put in the once shown below, and change them later if you like.
  • ROBOTS_DEBUG_DIR = c:\temp\debug
  • ROBOTS_OUT_DIR = c:\temp\out
  • ROBOTS_EXPECTED_DIR = c:\temp\expected
  • ROBOTS_LOGGING_FILE = testSuiteLog.properties
